# Contributing

Thanks for helping improve Web Resources. Contributions should keep the collection useful, focused, and easy to browse.

## What to contribute

Recommend resources that:

- Are useful for web design, development, accessibility, performance, tooling, or a closely related topic.
- Provide clear value beyond a basic search result.
- Are available through a stable, working URL.
- Have been personally evaluated or used by you.
- Are not already included in the collection.

Free and paid resources are both welcome, but paid products should provide clear value and must not be submitted as promotional spam.

## Adding a resource

1. Fork the repository and create a branch for your change.
2. Add the resource to the most relevant section in `README.md`.
3. Use the following format:

   ```md
   - [Resource Name](https://example.com/) — A concise description of what it provides and why it is useful.
   ```

4. Add a new category only when the resource does not fit an existing one.
5. Open a pull request explaining why the resource belongs in the collection.

## Writing guidelines

- Keep descriptions concise, factual, and free of marketing language.
- Start descriptions with a capital letter and end them with a period.
- Use the resource's official name and canonical URL.
- Avoid tracking parameters, referral links, and URL shorteners.
- Submit one resource per pull request when possible.
- Check spelling, grammar, and Markdown formatting before submitting.

## Pull requests

Your pull request should include:

- The resource name and URL.
- The category where it was added.
- A short explanation of why you recommend it.

Submissions may be edited, moved to another category, or declined to maintain the quality and scope of the collection.
